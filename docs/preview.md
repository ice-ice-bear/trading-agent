# 🚀 KIS 자율형 멀티 에이전트 트레이딩 시스템 (v2.0) 업데이트 프리뷰

기존의 대화형 투자 정보 검색 서비스를 넘어, **5개의 특화 AI 에이전트가 협업하는 자율형 모의투자 플랫폼**으로 진화합니다. 본 문서는 시스템 고도화 로드맵과 핵심 운용 엔진의 상세 메커니즘을 안내합니다.

---

## 🤖 1. 멀티 에이전트 오케스트레이션 (Multi-Agent Orchestration)

본 플랫폼은 투자 프로세스의 각 단계를 전담하는 5명의 AI 에이전트가 상호 유기적으로 협력하여 운용됩니다.

### 🔍 자산 건전성 모니터링 에이전트 (Portfolio Monitor)

* **역할:** 실시간 포트폴리오 익스포저(Exposure) 및 손익 변동성 추적
* **메커니즘:** 장중 5분 주기로 KIS API 연동을 통해 평잔, 실현/미실현 손익을 정밀 모니터링하며, 포트폴리오 스냅샷을 생성하여 데이터 기반 의사결정의 기초를 제공합니다.

### 📡 시장 모멘텀 스캐닝 에이전트 (Market Scanner)

* **역할:** 시장 데이터 분석 기반 투자 시그널 적합성 판별
* **메커니즘:** 기술적 지표(거래량 급증, 변동성 등)와 시장 심리를 결합하여 유망 섹터 및 종목을 발굴합니다. 분석 결과는 Claude AI를 통해 고도화된 매수/매매 시그널로 정제됩니다.

### ⚡ 알고리즘 매매 실행 에이전트 (Trading Executor)

* **역할:** 주문 집행 최적화 및 체결 결과 관리
* **메커니즘:** 리스크 거버넌스 에이전트의 최종 승인을 획득한 시그널에 대해 즉각적으로 주문을 집행합니다. 시장가 및 지정가 주문을 상황에 맞게 운용하여 최적의 체결 가격(Price Improvement)을 지향합니다.

### 🛡️ 리스크 거버넌스 에이전트 (Risk Manager) — **[Core Engine]**

* **역할:** 전방위 리스크 통제 및 자산 보호 정책 강제
* **메커니즘:** 모든 매매 요청은 본 에이전트의 검증을 통과해야 합니다.
  * **자본 배분:** 개별 종목 비중을 총 자산의 20% 이내로 제한
  * **손절매 정책(Stop-Loss):** 개별 포지션 -3% 도달 시 즉시 시장가 매도 집행 (무조건적 강제)
  * **집중 리스크 통제:** 동시 보유 포지션 수를 최대 5개로 제한하여 분산 투자 원칙 준수

### 📊 투자 성과 분석 리포터 (Report Generator)

* **역할:** 투자 성과 정량적 분석 및 전략 피드백 생성
* **메커니즘:** 일간/주간 단위로 운용 성과를 집계하여 샤프 지수(Sharpe Ratio), 최대 낙폭(MDD), 승률(Win Rate) 등 핵심 지표가 포함된 심층 리포트를 생성합니다.

## ⚙️ 2. 운용 정책 모델링 (Execution Policy)

사용자는 투명한 거버넌스 체계를 위해 에이전트의 운용 정책을 선택할 수 있습니다.

* **자율 매매 모드 (Autonomous):** 리스크 관리 엔진의 검증을 통과한 시그널에 대해 시스템이 전권을 가지고 즉각 매매를 집행합니다. 장중 대응이 어려운 전문 투자자에게 최적화된 모드입니다.
* **투자 권고 모드 (Advisory):** AI가 생성한 시그널을 사용자에게 대시보드로 통지하고, 최종 승인 시에만 주문을 집행합니다. AI의 분석력과 사용자의 직관을 결합하는 모델입니다.

## 📈 3. 고도화 로드맵 (Development Roadmap)

### 📍 Phase 1: 기반 인프라 확충 (Foundation)

* **핵심 과제:** 영속 저장소(SQLite) 기반 데이터 파이프라인 구축 및 자산 모니터링 엔진 활성화
* **산출물:** 실시간 자산 현황 대시보드(Portfolio View), 에이전트 활동 로그 시스템

### 📍 Phase 2: 트레이딩 자동화 (Automation)

* **핵심 과제:** KIS WebSocket 연동을 통한 실시간 시세 처리 및 자동 주문 시스템 고도화
* **산출물:** 실시간 워치리스트, 주문 관리 시스템(OMS), 시그널 중앙 통제 센터

### 📍 Phase 3: 분석 및 최적화 (Analytics & Optimization)

* **핵심 과제:** 고급 성과 지표 산출 엔진 및 리밸런싱 전략 구현
* **산출물:** 시각화 성과 분석 리포트, 전략 백테스팅 모듈, 위험 가치(VaR) 기반 리스크 관리

## 🔒 4. 보안 및 컴플라이언스 (Safety & Security)

본 시스템은 자산 운용의 안전성을 담보하기 위해 다층적인 보안 설계를 적용하였습니다.

1. **3중 모의투자 보호 체계:** 시스템 프롬프트, 클라이언트 오버라이드, 엔진 검증의 3단계 계층을 통해 실제 자산이 아닌 '모의투자(Demo)' 환경에서만 동작하도록 기술적으로 강제하였습니다.
2. **리스크 우선주의:** 모든 매매 시그널보다 리스크 관리 정책이 우선순위를 갖도록 설계되어, 비정상적인 시장 변동성 상황에서도 자산의 대규모 손실을 방지합니다.
3. **자격 증명 분리:** API Key 등 민감 정보는 메모리 및 환경 변수 기반으로만 관리되어 보안 유출 사고를 원천 차단합니다.
